## Project state

This is an early-stage Flutter project. `lib/main.dart` is still the default
`flutter create` counter demo — no app-specific screens, models, or state
management have been built yet. `pubspec.yaml` already declares the intended
stack (see below) but these packages are not yet wired up in code.

Package name: `ai_language_app`. Android applicationId / iOS bundle id still
use the default `com.example.ai_language_app` — update these before any real
release.

## Commands

Run all commands from the repo root (`ai_language_app/`).

```
flutter pub get                 # install/update dependencies after editing pubspec.yaml
flutter run                     # run on a connected device/emulator (or -d chrome/windows/etc.)
flutter analyze                 # static analysis / lints
flutter test                    # run all tests
flutter test test/widget_test.dart   # run a single test file
flutter test test/widget_test.dart --plain-name "Counter increments smoke test"  # run a single test by name
flutter build apk / ios / windows / macos / linux / web   # platform builds
```

There is no CI config, custom lint rules, Cursor rules, or Copilot
instructions in this repo yet — `flutter analyze` and `flutter test` are the
only checks currently enforced.

## Architecture notes

Since the app has no real structure yet, there isn't an established
architecture to follow — when adding features, set up the layout rather than
inferring one from existing code:

- `lib/main.dart` currently contains everything (app root + single screen).
  As real screens are added, split into a conventional `lib/` structure
  (e.g. `lib/screens`, `lib/widgets`, `lib/models`, `lib/providers` for
  Riverpod, `lib/router` for go_router) rather than growing `main.dart`.
- `flutter_riverpod` is the intended state-management approach — use
  providers rather than `StatefulWidget`-local state for anything beyond
  trivial UI state.
- `go_router` is the intended navigation approach — set up a central
  `GoRouter` configuration rather than Navigator 1.0 pushes.
- `flutter_secure_storage` is intended for any sensitive local data (e.g.
  API keys/tokens for whatever language-learning/AI backend this app talks
  to via `http`) — don't use `SharedPreferences` or plain files for secrets.
- The generated platform folders (`android/`, `ios/`, `linux/`, `macos/`,
  `windows/`, `web/`) are standard Flutter scaffolding; avoid hand-editing
  the generated plugin registrant files (`generated_plugin_registrant.*`,
  `generated_plugins.cmake`, `GeneratedPluginRegistrant.swift`) — these are
  regenerated by `flutter pub get`.
